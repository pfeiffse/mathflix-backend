## 🐍 Backend – wichtige Entscheidungen

### 1️⃣ **Flat‑Import‑Struktur**

Wir haben **bewusst KEIN `app.`‑Python‑Package mehr**.

✅ Richtig:

```python
from db import init_db
from models.user import User
from services.material_service import MaterialService
```

❌ Verworfen:

```python
from app.db import init_db
```

👉 Vorteil:

*   einfacher
*   weniger Python‑Importprobleme
*   klarere Struktur im Container

***

### 2️⃣ **Async SQLModel + SQLite (korrekt!)**

**db.py** (vereinfacht):

```python
engine = create_async_engine(
    "sqlite+aiosqlite:///./data/database.db"
)

async def init_db():
    async with engine.begin() as conn:
        await conn.run_sync(SQLModel.metadata.create_all)
```

✅ Wichtigste Erkenntnis:

*   `AsyncEngine` ≠ `create_all(engine)`
*   DDL **muss über `conn.run_sync()` laufen**

***

### 3️⃣ **Warum ein eigener `/data`‑Ordner?**

Problem:

    sqlite3.OperationalError: unable to open database file

Ursache:

*   SQLite braucht Schreibrechte
*   `/app` ist ein Code‑Volume → problematisch

Lösung:

```yaml
volumes:
  - ./data:/app/data
```

👉 **Best Practice für Container + SQLite**

## ⚡ FastAPI Startup‑Logik (final)

```python
@app.on_event("startup")
async def on_startup():
    await init_db()
    EventBus.add_middleware(log_event)
    EventBus.add_middleware(persist_event)
```

✅ Wichtige Fixes:

*   `FastAPI` korrekt importiert
*   `init_db` **async korrekt aufgerufen**
*   alle Abhängigkeiten explizit importiert

## 🐳 Docker-Setup (Kurzfassung)

### Backend

*   wird per `uvicorn main:app --host 0.0.0.0`
*   Port **8000 nur intern** (`expose`)
*   Startet stabil (kein Restart‑Loop mehr)

### Frontend

*   reines NGINX‑Image
*   serve statische Dateien

### NGINX (Reverse Proxy)

*   `/` → Frontend
*   `/api/*` → `backend:8000/*`
*   **keine CORS‑Probleme**
*   bereit für HTTPS

### docker-compose Prinzip

*   **klar getrennte Services**
*   **ein Einstiegspunkt**
